# Citation Format

## 行内引用

数字、市占率、增长率、定价等事实型陈述后立即加 `(¹)`、`(²)` 编号，与 `sources` 数组的 `id` 一一对应。

```
中国新茶饮市场 2024 年规模约 ¥3,547 亿元(¹)，同比增长 23%(²)。
```

## sources 数组格式

```json
{
  "id": 1,
  "title": "美团 2024 茶饮赛道白皮书",
  "url": "https://example.com/report",
  "publishedAt": "2024-12-01",
  "accessedAt": "2025-01-15",
  "domain": "example.com",
  "kind": "report|news|official|wiki|forum"
}
```

- `publishedAt` 不可推断时填 `null`。
- `accessedAt` 必填，工具会自动填当前日期。
- 同一 URL 不重复编号。

## 引用密度

- 任何**事实型**陈述（数字、命名、时间）必填引用。
- **解读型**陈述（"这表明 ..."）不必引用，但要确保被引用的事实陈述支撑得起这个解读。
- 一个段落里的同一 source 可省略后续 `(¹)`，只在第一次出现时标。

## 反例

```
据估算，X 占了 30% 的市场。      ← ❌ 无引用
Y 公司 CEO 表示 ...              ← ❌ 无引用
该品类增速放缓                    ← 解读句，无须引用，但前文必须有引用支撑这个判断
```
